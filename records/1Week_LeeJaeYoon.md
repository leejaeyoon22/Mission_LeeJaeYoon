### N주차 미션 요약

---

**[접근 방법]**

각 단계별로 요구하는 사항을 if문을 통해서 구별하고 그 안에 각 기능들을 구현하려고 하였다.<br>
그리고 8단계까지의 기능구현을 우선적으로 완료한 후, 전체적인 리팩토링을 진행하였다.<br><br>

- 1단계
<br> 명령을 입력 받는 부분 이후로 기능을 구현하지 않으면 자동적으로 종료 되기 때문에 scanner를 통해 명령을 입력받는 부분까지 구현하였습니다.<br><br>

- 2단계
<br> 입력 받은 명언과 작가를 String 변수를 받아서 저장시켜줍니다.<br><br>
- 3단계
<br>명언번호를 저장할 int 변수를 생성해줍니다.<br><br>
- 4단계
<br> 증감연산자를 이용하여 등록이 진행될 때마다 번호를 증가시켜줍니다.<br><br>
- 5단계
<br>번호와 명언과 작가를 저장할 객체를 생성한 후, 감소하는 for문을 통해 요구사항을 만족시켜줍니다.<br><br>
- 6단계
<br>startsWith 함수로 명령어를 구분하고 split을 통해 id값을 받아서 해당 id값을 객체와 비교하여 명언을 삭제합니다.<br><br>
- 7단계
<br>boolean 변수를 통해 삭제 동작 여부를 판단합니다. <br><br>
- 8단계
<br>6단계와 마찬가지로 조건문으로 객체를 판별한 후, 명언 수정을 진행합니다.<br><br>
- 9단계
<br>BufferedWriter 함수를 이용하여 종료하기 전에 객체를 txt파일에 저장한 뒤, 다시 프로그램을 실행할 때 txt파일을 load해온다.<br><br>
- 리팩토링
<br>완성한 코드들을 기능별로 구분하여 함수를 만들어서 옮겨줍니다. 그리고 중복되는 부분도 최대한 줄이기 위하여 마찬가지로 함수를 만들어서 적용시켜줍니다.

<br>
**[특이사항]**

Iterator를 쓴 부분도 리팩토링이 가능할 것 같은데 해결하지 못하였습니다.<br><br>
  **참고: [Refactoring]**

    - Refactoring 시 주로 다루어야 할 이슈들에 대해 리스팅합니다.
    - 1차 리팩토링은 기능 개발을 종료한 후, 스스로 코드를 다시 천천히 읽어보면서 진행합니다.
    - 2차 리팩토링은 피어리뷰를 통해 전달받은 다양한 의견과 피드백을 조율하여 진행합니다.
